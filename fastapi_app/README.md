# Expense Tracker API

This repository contains a simple FastAPI backend for a personal finance
application inspired by the UI shown in the provided images. The API
supports user registration and login, transaction management (list and
create) with pagination and search, and summary analytics.

## Features

- **User registration and login** – Basic endpoints for creating an
  account and logging in. Passwords are hashed using `bcrypt` via
  passlib if installed, falling back to a simple SHA256 implementation.
- **Transactions** – Endpoints to create income/expense transactions
  associated with a category and a user. The listing endpoint supports
  pagination (`page` and `size` query parameters) and optional search
  across notes and category names.
- **Summary** – An endpoint that returns aggregated totals for income
  and expense and a breakdown of spending by category within a date
  range. Defaults to the current month if no dates are provided.
- **Database** – Uses SQLAlchemy with a declarative model layer. The
  default connection is an in‑memory SQLite database for easy testing.
  Override the `DATABASE_URL` environment variable to use a PostgreSQL
  database, for example:

  ```bash
  export DATABASE_URL="postgresql+psycopg2://user:password@localhost:5432/financedb"
  ```

## Running the application

Install dependencies (FastAPI, Uvicorn, SQLAlchemy, Passlib). Then
start the server:

```bash
pip install fastapi uvicorn sqlalchemy passlib[bcrypt]
uvicorn fastapi_app.main:app --reload
```

Visit `http://127.0.0.1:8000/docs` to explore the interactive API
documentation generated by FastAPI.

## Example usage

Register a user:

```http
POST /auth/register
Content‑Type: application/json

{
  "username": "alice",
  "password": "secret"
}
```

Login:

```http
POST /auth/login
{
  "username": "alice",
  "password": "secret"
}
```

Create a category (not exposed via a route; insert directly into the
database) and then create a transaction:

```http
POST /transactions?user_id=1
{
  "type": "expense",
  "amount": 45.50,
  "category_id": 1,
  "note": "Groceries"
}
```

List transactions for user 1 (page 1, 10 per page, search term "shop"):

```http
GET /transactions?user_id=1&page=1&size=10&search=shop
```

Get summary for the current month:

```http
GET /summary?user_id=1
```

Get summary for a custom date range:

```http
GET /summary?user_id=1&start_date=2024-01-01&end_date=2024-12-31
```

## Notes

- A real application would implement proper authentication (e.g. OAuth2
  with JWT tokens) instead of passing `user_id` as a query parameter.
- For brevity, category management endpoints (create/read/update/delete)
  are not exposed via the API but can easily be added using the
  patterns established here.
- To initialise a PostgreSQL schema, set `DATABASE_URL` appropriately
  and run the app; SQLAlchemy will create the tables on startup.
